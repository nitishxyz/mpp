# `Fetch.polyfill`

Replaces the global `fetch` with a payment-aware wrapper.

## Usage

```ts twoslash
import { Fetch, tempo } from 'mpay/client'
import { privateKeyToAccount } from 'viem/accounts'

const account = privateKeyToAccount('0x...')

Fetch.polyfill({
  methods: [tempo.charge({ account })],
})

// Global fetch now handles 402 automatically
const res = await fetch('https://mpp.dev/api/ping/paid')
```

### Per-call accounts

Instead of defining an account at instantiation, you can pass it on a per-request basis using the `context` option. This is useful when you need to use different accounts for different requests.

```ts twoslash
// @noErrors
import { Fetch, tempo } from 'mpay/client'
import { privateKeyToAccount } from 'viem/accounts'

Fetch.polyfill({
  methods: [
    tempo.charge(),
  ],
})

const res = await fetch('https://mpp.dev/api/ping/paid', {
  // [!code focus:start]
  context: {
    account: privateKeyToAccount('0x...'),
  },
  // [!code focus:end]
})
```

## Return type

```ts
void
```

## Parameters

### methods

- **Type:** `readonly MethodIntent.Client[]`

Array of payment methods to use for handling `402` responses.

```ts twoslash
import { Fetch, tempo } from 'mpay/client'
import { privateKeyToAccount } from 'viem/accounts'

const account = privateKeyToAccount('0x...')

Fetch.polyfill({
  // [!code focus:start]
  methods: [tempo.charge({ account })],
  // [!code focus:end]
})
```
